SonED2 - Sonic 1 Rotating Special Stage Project Format


This type of project allows for editing the layouts of Sonic 1's Rotating Special Stages. It uses the following format:

  • [Project Type and Object Def reference]
    • Type:
    • Object Def:
  • [Map Layout]
    • Layout Compression:
    • Layout X Size:
    • Layout Y Size:
    • Layout Data:
  • [Player Start Positions]
    • Num Start Positions:
      • Start Position ?:
  • [Palettes]
    • Number of files:
      • Start index:
      • Number of entries:
      • File:
  • [Game Name]
    • Game Name:

    [Project Type and Object Def reference]
      This section contains the following general setup data:

      Type:
        The first line in any Project File is the "Type" identifier. This Project's "Type" should read "S1SS"

          Type: S1SS

      Object Def:
        This is the path and filename of the Sonic 1 Special Stage Map Object Listing file that contains graphic and animation information for displaying the "objects" that make up the level. This filename must not be accompanied by an extension; the extension ".lst" will be added by SonED2 when loading the list, and this filename will also be used to generate the filename string for the accompanying graphics file

          Object Def: ObjDef\ssobj


     [Map Layout]
      This section specifies how to load the map layout data

      Layout Compression:

        This entry specifies the type of compression that is used on the Map Layout data

          Layout Compression:  2

      Layout X Size:
      Layout Y Size:

        These entries specify the X and Y grid size of the Special Stage Map Layout

          Layout X Size:       64
Layout Y Size:       64

      Layout Data:

        This entry specifies the path and filename of the file containing the Map Layout data

          Layout Data:         ..\SStage\1.bin


    [Player Start Positions]
      This section specifies how to load the Player Start Positions data. The graphics that represent these positions within the editor are loaded by the Sonic 1 Special Stage Object Listing File

      Num Start Positions:
        This entry specifies the number of Player Start Positions that should be loaded. The value entered here determines the number of entries that should follow:

          Num Start Positions:   1

        Start Position ?:
          This entry specifies the path and filename of the file containing the Player Start Position data

            Start Position 0:    ..\SStage\Start\1.bin


    [Palettes]
      This section specifies how to load the Palette Color Data. The Genesis/Megadrive palette only has 64 entries, but an additional color space of 64 is available in the editor for creating "underwater" and other accessory palettes that can be flip/flopped with the primary palette during editing

      Palette data is typically stored in pieces, such as the usual separation between the "Main" palette section (player and global objects, indexes 0-15, usually unchanging) and the "Level" palette section (indexes 16-63, different for every level), so, generally more than one file will need to be loaded at a time

      Number of files:
        This entry specifies the number of Palette Color Data files to load

          Number of files: 2

        The data for loading each palette section is listed immediately below in the following format:

        Start index:
          This entry specifies where to place the incoming data in the palette. The value entered here is the palette index into which the data for the first color will be placed

            Start index:        0

        Number of entries:
          This entry specifies the number of palette color entries that should be loaded from the file

            Number of entries: 16

        File:
          This entry specifies the path and filename of the file containing the Palette Color Data

            File:              ..\Palettes\Sonic.bin


     [Game Name]
      This section allows a Game Name to be given to the Project, which will be displayed in the SonED2 titlebar along with the path and filename of the Project File

      Game Name:
        This entry is a text string that names the game that this Project File was created for

          Game Name: Sonic 1 Special
